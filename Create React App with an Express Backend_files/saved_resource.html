                                                                          The proxy isn't smart at all, it just forwards the requests along to the backend Express (or whatever) server. If there are any issues with routes not matching, 404 errors, etc, it's because something is wrong in the route setup on the Express side, or the UI is calling the wrong route.

                                                                          If that happens, the easiest way to debug it is to check the browser console's network tab, verify that the URL being called looks right. If not, fix it in the UI code. If it's right, then look at the backend routes.

                                                                                                • Dave Ceddia Mod Uday Saini • 3 months ago

                                                                                                  A very common misconception is that React handles HTTP requests -- it does not. In this example, we're using the fetch library for that (this line: fetch('/users')). In other examples you might see the "axios" library or some other library. In any case, those libraries are NOT bundled with React.

                                                                                                  The main reason this matters, is that when you're searching for help, you should look for "set headers with fetch" or "set headers with axios" rather than "set headers with react" (because React is a view library, and it has no HTTP functionality).

                                                                                                  Likewise for CORS: if you need to set CORS headers and such, that'll be done with your HTTP library of choice.

                                                                                                                          Maybe you already have an Express backend but you're rewriting the frontend in React and don't want to mess with webpack-dev-server and all that setup.

                                                                                                                          There are always multiple ways to do things :)
